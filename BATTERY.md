# バッテリー消費予測

## ハードウェア前提

| 項目 | 値 |
|------|-----|
| デバイス | M5Stack CoreInk (ESP32-PICO-D4) |
| バッテリー | 390mAh (内蔵LiPo) |
| センサ | Qwiic土壌センサ × 2 |
| 電源管理 | RTC timerSleep + powerOff |

## 1回の計測サイクル

| フェーズ | 消費電流 | 時間 |
|---------|---------|------|
| ESP32起動 + センサ初期化 | ~60-80mA | ~1s |
| baseline安定待ち (ポーリング) | ~60-80mA | ~0.4-0.8s |
| E-Paper描画 (epd_quality) | ~30-50mA | ~3-4s |
| **小計 (計測+表示)** | **~70mA平均** | **~5.5-6s** |
| BLE advertising | ~55mA平均 | ~3.8s |
| **合計** | **~65mA平均** | **~9.5s** |

→ 1回あたり約 **0.17mAh** (BLEなしなら0.11mAh)

スリープ時: **~2.6µA** (RTC timerSleep/powerOff状態)

※ BLE advertising込み (0.17mAh/cycle) で計算。

## BLE advertising 追加時の影響

BLE advertisingでセンサ値をブロードキャストし、ゲートウェイ（ラズパイ等）経由でLINE連携する構成を想定。

### BLE追加分の消費

実装: `ADV_TYPE_NONCONN_IND`, 20-40ms間隔, 3秒間broadcast

| フェーズ | 消費電流 | 時間 |
|---------|---------|------|
| BLEスタック初期化 (controller + bluedroid) | ~80mA | ~500ms |
| ADVデータ設定 + コールバック待ち | ~80mA | ~150ms |
| advertising (20-40ms間隔, 3秒間) | ~50mA平均 (TX時130-180mA, idle時~40mA) | 3s |
| BLEスタック解放 (disable + deinit) | ~60mA | ~200ms |
| **BLE追加分合計** | **~55mA平均** | **~3.8s** |

→ 1サイクルあたり **+0.06mAh**（0.11 → 0.17mAh）

※ 20-40ms間隔でも3s中のTX時間は計~300msで、大部分はdelay()中のidle（~40mA）。
広告間隔が短い分、平均電流が低く、3秒に延びてもエネルギー消費は小さい。

### 通信方式の比較

| 方式 | 追加消費/回 | 特徴 |
|------|-----------|------|
| BLE advertising | +0.06mAh | 低コスト。ゲートウェイが必要 |
| BLE connection | +0.2-0.5mAh | 接続待ち時間が長く非推奨 |
| WiFi (直接LINE送信) | +0.5-1.0mAh | ゲートウェイ不要だがバッテリー寿命が大幅に短縮 |

BLE advertisingならバッテリーへの影響は許容範囲。WiFi直接送信は避けるべき。

## 備考

- BLE advertisingはsleep前に3秒間のみ有効化し、完了後にスタックを完全解放
- E-Paperは表示維持に電力不要（powerOff後も画面保持）
- LiPo自己放電は月3-5%（~0.4-0.65 mAh/日）で計算
- スリープ消費(2.6µA)はどの間隔でも0.06mAh/日で一定かつ無視できるレベル
- 上記は理論値。実測で±30%程度の誤差あり
